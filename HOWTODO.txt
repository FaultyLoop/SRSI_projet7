LEXIQUE :
username:	Nom de l'utilisaeur pour les scripts (srsi7 par defaut)
ip-chain:	Ip(s) de serveur de chaine (n'importe le quel tant qu'il est configuré)
ip-files:	Ip de server de fichier
SN	: 	Side Note
AC	:	After Command


------- PREPARATION DE L'ENVIRONNEMENT -------
==== Serveur de fichier ====
1.	useradd <username> -m
2.	su <username>
3.	mkdir ~/files
4.	chmod 0700 ~/files

==== Serveur de chaine ===
1.	useradd <username> -m
2.	su <username>
3.	##COPIE DU SCRIPT chaincheck.sh dans ~/##
4.	chmod 0700 ~/chaincheck.sh
5.	~/chaincheck.sh
6.	##EDITER ~/chaindb/fileserver ajouter les ipv4 des serveurs de fichier DÉDIÉ au serveur de chaine (127.0.0.1 correspond à toutes les ipv4 des interfaces locale)

==== Server Host des scipts ====
1.	useradd <username> -m
2.	su <username>
3	ssh-keygen
4.	ssh-copy-id <ip-chain>
SN.	Autant que voulu ↑
5.	ssh-copy-id <ip-files>
SN.	Autant que voulu ↑
6.	##COPIE DES SCRIPTS split.sh, shuffle.sh, recover.sh et join.sh dans ~/ ##
7.	./shuffler.sh -c <ip-chain> --make-list 
7 alt.	Crée/éditer le fichier ~/chainip ajouter les ipv4 des serveurs de chaine (1 ou plusieurs) 

------- TEST DE L'ENVIRONNEMENT -------
==== Server Host des scipts ====
1.	echo /proc/cpuinfo > ~/infoscpu.txt
SN.		↑ Ou autre patterne reconnaissable

AC.	ls dir_*
